Interface Documentation

Overview

The ROS Interface class provides a connection between ROS (Robot Operating System) environments and the KnowRob knowledge base.

Prerequisites

Before using the ROSInterface, ensure that the following requirements are met:

  • ROS Noetic is installed. (This version is tested and recommended for compatibility.

Actions Provided

ROSInterface provides several actions to interact with the KnowRob system. These actions are as follows:

1. AskAll Action

  • Action Name: /knowrob/askall
  • Purpose: Executes a query and retrieves all possible solutions.
  • Input: GraphQueryMessage which includes the query string and parameters.
  • Output: Returns all possible answers to the query.

2. AskOne Action

  • Action Name: /knowrob/askone
  • Purpose: Executes a query and retrieves the first valid solution.
  • Input: GraphQueryMessage as the query.
  • Output: Returns the first possible answer if available.

3. AskIncremental Action

  • Action Name: /knowrob/askincremental
  • Purpose: Provides a way to incrementally retrieve answers to a query.
  • Input: GraphQueryMessage containing the query details.
  • Output: Returns an queryId to identify, that can be used to fetch subsequent solutions using the askincremental_next_solution action.

4. AskIncrementalNextSolution Action

  • Action Name: /knowrob/askincremental_next_solution
  • Purpose: Fetches the next available solution for an ongoing incremental query.
  • Input: queryId to identify the query session.
  • Output: The next solution or a status indicating no more solutions are available.

5. Tell Action

  • Action Name: /knowrob/tell
  • Purpose: Submits knowledge to be asserted into the knowledge base.
  • Input: GraphQueryMessage which includes statements to be asserted.
  • Output: Status indicating success or failure of the operation.

Usage

To utilize the ROSInterface run:

roslaunch knowrob knowrob.launch
